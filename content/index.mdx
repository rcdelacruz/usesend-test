# useSend Testing Suite

Welcome to the **useSend Testing Suite** documentation! This comprehensive testing suite helps you verify that your self-hosted useSend instance is properly configured and working.

## What is useSend?

useSend is a self-hosted email service platform that provides:
- RESTful API for transactional emails
- SMTP server support
- Email analytics and tracking
- Integration with AWS SES

## Features

This testing suite provides:

- ✅ **REST API Testing** - Verify transactional email sending via REST API
- ✅ **SMTP Server Testing** - Test SMTP configuration (optional)
- ✅ **Configuration Validation** - Ensure all required settings are configured
- ✅ **Detailed Logging** - Color-coded output for easy debugging
- ✅ **Error Reporting** - Comprehensive error messages and troubleshooting
- ✅ **Email Delivery Tracking** - Monitor email status and delivery

## Quick Start

Get up and running in 3 simple steps:

```bash
# 1. Install dependencies
npm install

# 2. Configure your .env file
cp .env.example .env
# Edit .env with your useSend credentials

# 3. Run the tests
npm test
```

## Prerequisites

Before you begin, ensure you have:

- Node.js (v14 or higher)
- A self-hosted useSend instance
- useSend API key
- Verified email domain (in AWS SES)
- Verified recipient email (if in AWS SES sandbox mode)
